# Работа с AI-агентами

## Основной процесс

Каждая задача создаётся через GitHub Issues.

Рабочий цикл:

1. Создать issue.
2. Создать отдельную ветку от main.
3. Поручить агенту выполнить маленькую задачу.
4. Проверить изменения.
5. Сделать commit.
6. Открыть pull request.
7. Проверить pull request.
8. Объединить изменения в main.

## Правило

Одна задача — одна ветка — один pull request.

## Что нельзя делать

- не писать всё сразу в main;
- не давать агенту слишком большую задачу;
- не смешивать документацию, backend и приложение в одном задании;
- не принимать изменения без проверки.
